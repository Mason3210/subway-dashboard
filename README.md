# 全球轨道交通安全看板 (Global Subway Safety Dashboard)

一个专门为地铁安全工程师设计的综合性安全数据监控和分析平台。

## 功能特点

### 🚇 安全监控
- 全球地铁安全事件实时监控
- 各地区安全评分对比分析
- 安全事件趋势分析
- 合规性状态跟踪

### 📊 数据可视化
- 交互式图表和仪表盘
- 地区安全评分对比图
- 实时数据更新显示
- 多语言支持（中文/英文）

### 🔄 自动化数据采集
- 每日定时数据爬取（早8点）
- 多数据源整合
- 智能内容摘要生成
- 数据质量评估

### 🌍 数据源
- 国际公共交通协会 (UITP)
- 联邦运输管理局 (FTA)
- 中国地铁运营协会
- 其他权威轨道交通机构

## 技术架构

- **后端**: Python Flask
- **前端**: HTML5, CSS3, JavaScript, Bootstrap 5
- **数据可视化**: Chart.js
- **数据爬取**: Beautiful Soup, Requests
- **任务调度**: Schedule

## 快速开始

### 1. 环境准备

确保已安装 Python 3.7+，然后安装依赖：

```bash
pip install -r requirements.txt
```

### 2. 启动仪表盘

```bash
python app.py
```

访问 `http://localhost:5000` 查看仪表盘

### 3. 运行数据采集

测试数据采集功能：

```bash
python daily_updater.py
```

## 使用说明

### 仪表盘功能

1. **关键指标**: 显示全球安全事件总数、平均安全评分等核心指标
2. **地区对比**: 对比不同地区的地铁安全评分
3. **数据源状态**: 显示各个数据源的连接状态
4. **最新动态**: 显示最新的地铁安全相关新闻和更新

### 数据采集

系统会自动在每天早上8点执行数据采集任务，包括：
- 访问配置的权威网站
- 提取安全相关信息
- 生成内容摘要
- 更新仪表盘数据

### 手动更新

点击右下角的"刷新数据"按钮可以手动触发数据更新。

## 配置文件

### 数据源配置

在 `scraper.py` 中配置目标网站：

```python
self.target_sites = [
    {
        'name': '数据源名称',
        'url': '网站URL',
        'search_paths': ['/news', '/safety'],
        'language': 'zh'
    }
]
```

### 定时任务

在 `daily_updater.py` 中配置定时任务：

```python
# 修改定时时间
schedule.every().day.at("08:00").do(self.run_daily_scraping)
```

## 扩展功能

### 添加新的数据源

1. 在 `scraper.py` 的 `target_sites` 列表中添加新网站配置
2. 根据需要调整选择器和关键词
3. 测试数据采集功能

### 自定义图表

修改 `static/dashboard.js` 中的 `updateRegionalChart` 函数来自定义图表样式和类型。

### 添加新的安全指标

1. 在后端 `app.py` 的 `SafetyData` 类中添加新指标
2. 在前端页面中添加对应的显示组件
3. 更新 JavaScript 数据处理逻辑

## 故障排除

### 数据采集失败

1. 检查网络连接
2. 验证目标网站是否可访问
3. 检查网站结构是否发生变化
4. 查看日志文件 `daily_scraper.log`

### 仪表盘显示异常

1. 检查浏览器控制台错误信息
2. 验证 API 响应是否正常
3. 检查数据格式是否正确

### 性能优化

1. 对于大量数据，考虑实现分页显示
2. 使用缓存机制减少重复数据请求
3. 优化数据库查询（如果使用数据库存储）

## 安全注意事项

1. 遵守目标网站的 robots.txt 规则
2. 设置合理的请求间隔，避免对目标网站造成负担
3. 对敏感数据进行适当的脱敏处理
4. 定期更新依赖包以修复安全漏洞
